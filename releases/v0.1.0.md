## Summary
- Initial beta release of Gonvex, a Convex-style backend platform using Go, Postgres, and TypeScript with realtime WebSocket subscriptions
- Publish five npm packages under the `@gonvex` organization scope (protocol, client, react) plus the `gonvex` CLI and `create-gonvex` initializer
- Realtime dashboard with tenant-aware task grids, fuzzy search, email-based authentication, and S3-compatible file storage
- Developer tooling including production Dockerfiles, release automation, and a Vite React starter template

## Changes
- **npm Packages**: Scoped libraries (`@gonvex/protocol`, `@gonvex/client`, `@gonvex/react`) and CLI tools (`gonvex`, `create-gonvex`) under the `@gonvex` org; configured publishable tarballs with built JS and TypeScript declarations
- **Realtime Dashboard**: Added live task grid with composite indexes (active/workspace/status/priority/assignee/due) and trigram indexes for fuzzy search across name, title, and description; restored the Test page with tenant/database picker for multi-tenant debugging
- **File Storage**: Implemented S3-compatible storage backend supporting MinIO and AWS S3, including dashboard file APIs and a migration tool to clone Convex file-storage blobs into the new backend
- **Authentication**: Replaced basic auth with session-based login; added email authentication provider, tenant-aware auth context, and configurable auth providers (Google disabled by default)
- **Developer Experience**: Added production Dockerfiles for the runtime and dashboard, npm release automation scripts, and a complete Vite React starter template with generated bindings

## Technical Notes
- `@gonvex/*` packages now publish only the `dist/` directory with corrected `main`, `types`, and `exports` fields; previously `types` pointed to unpublished `src/`
- Release validation scoped to the five publishable packages to prevent unrelated apps/docs typecheck errors from blocking npm releases
- Schema parsing recognizes `TrigramIndex` and emits deterministic, sorted `api.ts` codegen
- `cmd/clone-storage` migrates Convex file-storage blobs into S3/MinIO and upserts matching `_gonvex_files` rows so cloned attachments resolve correctly
- Dashboard grid resolves columns dynamically against both normalized tenant schemas and legacy flat mirror tables for backwards compatibility
