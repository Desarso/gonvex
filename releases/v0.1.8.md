## Summary
- **Data engine & sandbox**: New DuckDB-powered data file support (CSV/XLSX/XLS) with ingest, query, and profiling APIs, plus a Go sandbox runner for secure code execution within actions.
- **Cron scheduler & observability**: Built-in cron scheduler (`app.Cron`, `ctx.Scheduler`) with a real-time Health Overview dashboard showing function metrics, cache rates, and scheduler lag.
- **Browser query cache**: Transparent client-side query caching with automatic invalidation, plus fixes for live query updates from scheduled jobs and actions.
- **Error tracking**: Lightweight browser SDK reporting and dashboard triage for grouped client errors, occurrence history, status, and assignment.
- **Dashboard collaboration**: Project membership controls, notifications tray for invites, and project-scoped environment variables injected into function contexts.

## Changes

**Data & Analytics**
- Added DuckDB data-file engine for server-side processing of CSV, XLSX, and XLS files with SELECT-only validation and automatic re-ingestion on cache miss.
- New Data API for ingest/inspect/query/profile operations available via `RuntimeContext`.
- Go sandbox runner with whitelisted host calls for secure code execution; supports dataset, tool, policy, and result abstractions.
- Storage proxy serves files through the runtime's public URL with HMAC-signed URLs instead of exposing internal S3 endpoints.

**Scheduler & Background Jobs**
- In-process cron scheduler supporting standard 5-field cron expressions.
- New registration APIs: `app.Cron`, `app.CronExpr`, and runtime methods `ctx.Scheduler.RunAfter` / `RunAt`.
- Scheduled jobs and long-running actions now broadcast live-query invalidations; actions can push invalidations mid-execution via `ctx.NotifyTableChange`.
- Health Overview dashboard with real-time Recharts visualizations: function call rates, failure rates, cache hit rates, execution latency, scheduler lag, and concurrency-by-kind gauges.

**Dashboard & Project Management**
- Notifications tray surfaces unread membership invitations and project alerts with read-state management.
- Project-scoped environment variables configurable in the dashboard and injected into function runtime contexts (30-second TTL with cache invalidation on edit).
- Improved Files tab lists objects directly from storage (bypassing metadata table gaps), supports row selection, bulk download, and icon-button actions.
- Project creation supports database mode selection (single vs multi-tenant) and environment labeling via `GONVEX_ENVIRONMENT`.
- Dashboard authentication falls back to runtime auth when local credentials are unavailable; sessions persist across reloads with signed access tokens.

**Realtime & Caching**
- Transparent browser query caching with background refresh and stale-while-revalidate semantics.
- Fixed live query invalidation for scheduled jobs and action calls (previously only mutations broadcasted table changes).
- Fixed nil result serialization to return explicit JSON `null` instead of omitting the field, resolving stuck loading states in React hooks.

**Runtime Reliability**
- Serialized schema apply using PostgreSQL advisory locks to prevent "tuple concurrently updated" DDL races during concurrent syncs.
- Reuse existing tenant databases when hydrating persisted tenants instead of creating duplicates.
- Cached compiled Go plugins keyed by bundle hash to skip recompilation on restarts; configure cache directory via `GONVEX_PLUGIN_CACHE_DIR`.
- Per-project dev sync authentication: `/dev/sync` validates against the target project's registered key instead of a global shared secret.

**Error Tracking**
- Added an opt-in browser error reporter for uncaught exceptions, unhandled promise rejections, custom reports, release metadata, and user context.
- Added server-side fingerprinting, occurrence aggregation, status and assignee workflows, retention controls, and project-scoped error APIs.
- Added dashboard triage views for searching, filtering, inspecting, assigning, resolving, ignoring, and deleting grouped errors.

**CLI & Developer Experience**
- `gonvex dev --once` exits non-zero on sync failure for reliable CI/CD pipelines.
- Source bundles included in dev sync for hot-reload but omitted from generated production manifests to reduce bundle size.
- Dashboard Vite dev server pinned to port 5174 (`strictPort`) for consistent local development.
- Documentation refreshed for current CLI commands (`dev`, `init`, `create`, `env`) and the self-hosting deployment model.

## Technical Notes
- **npm packages**: `@gonvex/client` and `@gonvex/react` include query cache integration; `@gonvex/client` also exports the browser error-reporting API. Both features include regression coverage.
- **Go runtime**: Requires CGO enabled for DuckDB integration and plugin compilation; new `GONVEX_DATA_DIR` volume for data-file artifacts.
- **Schema migrations**: Advisory locks serialize DDL per-database; schema fingerprints skip redundant applies to reduce database load during hot-reload.
- **Auth**: Dashboard project endpoints scoped to runtime-accessible users; runtime admin key allowed across project routes.
- **Storage**: Files served via runtime proxy when `GONVEX_PUBLIC_URL` is configured; MinIO/S3 remains internal.
- **Release process**: Fixed npm release version selection to prevent stale tags from producing lower package versions.
