## Summary

- **CLI**: TypeScript manifest generator now preserves function dependency metadata (`Reads`, `Writes`, `ShareByPermissions`, columns, filters, etc.) for optimized reactive query invalidation.
- **Runtime**: Go query handlers can access reactive change revision context to safely coalesce common base reads across identity-scoped subscription groups.
- **Database**: Process-wide PostgreSQL connection budget is now configurable up to 64 connections (safety ceiling) with new observability metrics for wait times and pressure.
- **Load Testing**: `gonvex-load` runner supports injection of mutations during steady-state hold, measuring mutation acknowledgement and reactive invalidation latency.

## Changes

### CLI and Schema
- The `@gonvex/cli` package parses dependency declarations (e.g., `gonvex.Reads("table").Columns("col").Filters("filter")`) and includes them in the generated function manifest.
- Added test coverage for dependency metadata preservation in the dev manifest.

### Runtime and Database
- Added `QueryChangeInfo` context API (`WithQueryChange`, `QueryChange`) allowing Go handlers to detect reactive invalidation waves and avoid redundant base reads.
- Increased maximum configurable PostgreSQL connection budget from hardcoded 20 to a ceiling of 64 via `GONVEX_DB_MAX_TOTAL_CONNS`.
- Exposed process-wide connection budget statistics (active, waiters, wait count, wait duration) through `RuntimeBudgetStats()`.

### Load Testing and Observability
- `gonvex-load` supports `--mutation-path`, `--mutation-rate`, and `--mutation-args` flags to drive write workloads and measure end-to-end reactive delivery.
- Runner reports mutation latency, server duration, and invalidation message counts (full results, patches, progress).
- Added comprehensive load test documentation covering connection budget tuning and query amplification analysis.

## Technical Notes

- **npm CLI**: `@gonvex/cli` v0.1.13 parses chained dependency options (`Columns`, `Filters`, `OrdersBy`, `Windowed`, `Predicate`) from Go source files during manifest generation.
- **Runtime**: `QueryChange` context values include change reason and timestamp; zero values indicate initial/direct queries.
- **Database**: Environment variable `GONVEX_DB_MAX_TOTAL_CONNS` now accepts values up to 64; the default remains 20. Budget wait metrics are exposed separately from per-pool SQL statistics.
- **Load Runner**: Mutation arguments support template variables (`${tenant}`, `${userId}`, `${sequence}`, `${mutationId}`). The runner waits for all mutation results before closing connections.
