## Summary
- **Realtime Connection Dashboard** – Monitor live websocket activity, active subscriptions, and user presence with per-connection browser, platform, and routing details.
- **Database Health Metrics** – View connection pool pressure, wait times, and query cache sources (Redis vs database) with new telemetry charts in the dashboard.
- **Multi-tenant Scale & Safety** – Tenant databases now use collision-free UUIDv6 physical names; schema rollout is parallelized and bounded by proxy timeouts to prevent provisioning storms.
- **Project Key Rotation** – Rotate project API keys with durable persistence and immediate runtime propagation without restarting services.

## Changes
- **Dashboard**
  - Added Realtime page showing active connections, subscription paths, user identity, and recent activity streams.
  - New Database Health section displaying pool occupancy, connection waits, and pressure indicators.
  - Fixed session restoration for Google Auth users and preserved project deep links during cold loads.
  - Improved project chooser scrolling and environment variable editor with full `.env` text view support.
  - Logs now show execution IDs, tenant context, and whether queries hit Redis or the database.

- **Performance & Scale**
  - Parallelized tenant schema rollout with timeout bounding; skip installation of unchanged realtime triggers.
  - Optimized schema inspection using fast PostgreSQL catalogs and batched queries.
  - Extended Redis cache TTL and added table-level query cache invalidation.
  - Enforced database connection safety ceilings to prevent exhaustion under load.

- **Reliability**
  - Hardened websocket reconnection and transactional invalidation edge cases.
  - Fixed runtime tenant connection storms and auth state preservation across subscription reruns.
  - React SDK now retries query errors after component remounts.

## Technical Notes
- **CLI**: Added `gonvex-load` command for persistent WebSocket stress testing and multitenant reactive load validation.
- **Packages**: `@gonvex/react` preserves auth across subscription reruns and retries failed queries after remounts; `@gonvex/client` supports cache invalidation by table.
- **Schema**: Tenant databases migrated to opaque UUIDv6 physical identifiers (legacy databases remain discoverable); schema synchronization skips unchanged trigger installs.
- **Runtime**: New connection pool telemetry requires `GONVEX_DB_MAX_OPEN_CONNS`, `GONVEX_DB_MAX_IDLE_CONNS`, and `GONVEX_DB_MAX_TOTAL_CONNS` environment variables (defaulting to 16/1/20).
- **Configuration**: `GONVEX_ROWS_CACHE_TTL` default increased to 10m; added `GONVEX_TENANT_LISTENER_LIMIT` and shared subscription grace settings for high-fanout workloads.
