## Summary
- The dashboard now shows the **Errors** section only for projects that have enabled error tracking; the client reporter automatically registers this capability on startup.
- Projects can be renamed from the dashboard’s General settings without changing their ID, database, or storage bucket.
- The CLI adds `gonvex env push FILE` to atomically replace a project’s server-side environment variables from a local file.
- Fixed tenant ownership so new projects no longer inherit unrelated legacy global tenants and require explicit tenant relationships.

## Changes
- **Error tracking visibility** — The dashboard sidebar and deep links now respect a per-project `errorTrackingEnabled` flag. The JavaScript reporter sends a registration request on initialization, and the first captured error envelope also enables the flag automatically so existing clients remain compatible.
- **Project renaming** — Added an inline name editor in General settings that PATCHes the project display name. The project ID, database, and storage relationships remain unchanged.
- **Bulk environment upload** — `npx gonvex env push .env.production` (also `env upload`) reads a local env file and replaces the entire remote variable set in one operation. The CLI refuses to upload files containing `GONVEX_PROJECT_KEY`, `GONVEX_DEPLOY_KEY`, or `GONVEX_KEY` to prevent leaking credentials into function environments.
- **Tenant ownership fixes** — New projects using UUIDv6 IDs no longer auto-inherit legacy global tenant databases. Tenant relationships must now be explicit, and deleting a project cleans up its associated tenant entries.

## Technical Notes
- **npm CLI**: `@gonvex/cli` adds `env push`/`upload` with optional `--file` support and new `dotEnvVariableNames` parsing. All workspace packages bumped to `0.1.9`.
- **Schema**: Project registry adds `error_tracking_enabled BOOLEAN DEFAULT FALSE` and a new `gonvex_runtime_tenants` table with foreign-key cascade delete. Project IDs and tenant relationship IDs now use UUID v6.
- **Runtime**: Project registry initialization uses a PostgreSQL advisory lock to prevent race conditions during concurrent schema creation. Project env endpoints accept either a dashboard session or the exact registered project key via `x-gonvex-key` / `Authorization: Bearer`.
- **Server**: Removed implicit `hydrateLandlordTenants` call from auth loading. Error tracking adds `POST /errors/register` and `GET /dev/errors/status` endpoints.
