## Summary
- Replaces Firebase dashboard login with native, project-scoped Google OAuth and adds CLI account sessions, personal access tokens, and project provisioning.
- Ships a hardened production deployment template with isolated services, automated runtime regression tests, ACME routing, and health-gated PostgreSQL backups.
- Adds dashboard account profiles with admin API key management, a compact theme toggle, and improved error-tracking compatibility detection.

## Changes
- **Authentication** — Replaces Firebase dashboard login with native, project-scoped Google OAuth across the CLI, runtime, and React SDK (`GonvexAuthProvider`, `GoogleSignInButton`, `useGonvexAuth`). Adds CLI account sessions (`whoami`, `logout`), personal access tokens, and runtime-backed project provisioning. Hardens project environment endpoints to require project-bound keys.
- **Production deployment** — Introduces `docker-compose.production.yml` for an isolated, authenticated stack with durable state, bounded resources, retained backups, and exact regression coverage. Adds a GitHub Actions workflow to validate the production Compose contract and run runtime tests. Isolates production service discovery with stack-specific names and health-gates PostgreSQL backups on shared networks.
- **Runtime hardening** — Routes production hosts through HTTP ACME, exposes the runtime on an isolated product edge with explicit public signed HTTP handlers, and isolates compiled plugins by runtime build to prevent ABI mismatches.
- **Dashboard UX** — Adds an account profile dialog for viewing role/credential reach and managing runtime-wide admin API keys. Replaces the text theme button with a compact icon toggle and streamlines the project switcher. Improves error tracking with availability states (Checking / Capturing / Unavailable) and clear messages for legacy runtimes. Fixes UUID project tenant isolation and errors navigation.

## Technical Notes
- Build pipeline now generates package declarations before the release typecheck.
- `Dockerfile.runtime` sets `GOMAXPROCS=2` and `GOFLAGS=-p=2`; `Dockerfile.dashboard` sets `UV_THREADPOOL_SIZE=2` and `RAYON_NUM_THREADS=2`.
- `.env.example` documents `GONVEX_PUBLIC_URL`, `GONVEX_AUTH_URL`, `GONVEX_GOOGLE_CLIENT_ID/SECRET`, `GONVEX_DASHBOARD_AUTH_PROJECT_ID`, and `GONVEX_TRUSTED_PROXY_CIDRS`.
- Removes the `firebase` dependency from the dashboard.
