# Quickstart

This page describes the intended user workflow for starting a Gonvex-backed app.

## Create An App

```bash
npm create gonvex@latest my-app
cd my-app
```

Explicit Vite React template:

```bash
npm create gonvex@latest my-app -- --template vite-react
cd my-app
```

The initializer should create:

```txt
my-app/
  app/ or src/
  gonvex/
    schema.go
    tasks.go
    _generated/
  package.json
  gonvex.json
  .env.local
```

## Start Development

```bash
npx gonvex dev
```

Framework templates usually wrap this in the app dev command. For Vite React:

```bash
npm run dev
```

which runs:

```bash
gonvex dev -- vite
```

`gonvex dev` should:

- start or connect to the local Gonvex runtime
- watch the `gonvex/` folder
- generate TypeScript bindings
- sync schema and function metadata to the runtime
- run your app dev server if configured

## Define Schema

```go
// gonvex/schema.go
package backend

import "github.com/gonvex/gonvex/pkg/gonvex"

func Schema(s *gonvex.Schema) {
  s.Table("tasks", func(t *gonvex.Table) {
    t.ID("id")
    t.String("title")
    t.String("status")
    t.Time("created_at")

    t.Index("by_status", "status")
    t.Index("by_created_at", "created_at")
  })
}
```

## Use Generated Bindings

```ts
import { api } from "./gonvex/_generated/api";
import { useMutation, useQuery } from "./gonvex/_generated/react";

export function Tasks() {
  const tasks = useQuery(api.tasks.list, { status: "open" });
  const createTask = useMutation(api.tasks.create);

  return <TaskList tasks={tasks ?? []} onCreate={createTask} />;
}
```

## Connect To A Runtime Project

For real projects, connect the repo to a Gonvex project:

```bash
npx gonvex login
npx gonvex project create my-product
npx gonvex link --project my-product
```

This writes project connection data to `gonvex.json` and local environment files.

For production, the browser points at the cloud runtime:

```txt
VITE_GONVEX_WS_URL=wss://my-product.gonvex.cloud/ws
```

## Add Tenants

If your app is multi-tenant:

```bash
npx gonvex tenant create acme --project my-product
npx gonvex tenant create globex --project my-product
```

Each tenant can route to its own database connection while sharing the same project code and generated bindings.
